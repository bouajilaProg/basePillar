### 1. feat(backend:schema) – Drizzle ORM & Auth Models
* **Database Setup:** Initialize Drizzle with the PostgreSQL driver.
* **Tables:**
    * `users`: `id`, `email` (unique), `password` (hashed), `createdAt`.
    * `organizations`: `id`, `name` (unique), `slug`, `createdAt`.
    * `memberships`: `userId`, `orgId`, `role` (enum: `admin`, `normalUser`).
* **Relations:** Define many-to-one relations between memberships and users/orgs.

### 2. feat(backend:auth) – Secure JWT & Cookies
* **Encryption:** Use `bcrypt` or `argon2` for password hashing before storage.
* **Transport:** * Implement JWT generation on login/register.
    * Set the JWT in an `HttpOnly`, `Secure`, `SameSite=Strict` cookie.
    * Implement a `Guard` to extract and verify the token from the `Cookie` header instead of the `Authorization` bearer.
* **Logic:** * `POST /auth/register`: Create user + Create unique Org + Assign `admin` role.
    * `POST /auth/login`: Verify hash + Issue Cookie.
    * `POST /auth/logout`: Clear the cookie.

### 3. feat(backend:swagger) – API Documentation
* **Config:** Use `@nestjs/swagger` in `main.ts`.
* **Security:** Configure Swagger to recognize the Cookie-based auth for testing endpoints in the browser.
* **Validation:** Use `class-validator` with `@ApiProperty` to ensure DTOs are documented.

### 4. feat(frontend:pages) – Flat UI Implementation
* **Design System:** Strict "no-card" policy. Use `divide-y`, `border-b`, and `spacing` for visual hierarchy.
* **Layouts:**
    * **Auth:** Minimalist centered forms with `Label` and `Input` from Shadcn.
    * **Org Dashboard:** Data tables and list items using raw borders.
* **State:** Use a `useAuth` hook that checks for a session via a `/me` endpoint (since JS cannot read the HttpOnly cookie).
